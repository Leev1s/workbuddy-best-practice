# Workflow: Write New Guide

## Purpose

为本仓库新增一篇可公开发布的 WorkBuddy 最佳实践文章。

## Steps

1. 阅读 `AGENTS.md`、`README.md`、`sources.md` 和目标目录 `README.md`。
2. 打开对应 WorkBuddy 官方文档和 Changelog。
3. 列出可写成官方事实的内容，单独列出实践解释。
4. 写正文，默认包含 TL;DR、Official facts、Practical interpretation、Best practices、Security / permission notes、Further Reading。
5. 检查 `.workbuddy/`、Claude Code、第三方 Skill / Connector 的边界表述。
6. 更新目标目录 README 或根 README 的导航。
7. 运行 Markdown 空文件、占位符正文、内部链接和 git status 检查。

## Output

- 新增或修改的 Markdown 文件。
- 必要的导航链接。
- Further Reading 官方来源。
