# Markdown and Asset Output Rules

## Markdown

- 中文为主，必要英文术语保留。
- 使用清晰标题层级：一个 H1，后续按 H2 / H3 展开。
- 每篇正文尽量包含：TL;DR、Official facts、Practical interpretation、How to use、Best practices、Common mistakes、Security / permission notes、Further Reading。
- 内部链接使用相对路径。

## Assets

- 仓库级多媒体资产放在 `assets/`。
- README 首屏优先引用 WebP 小图，保留 PNG 原图用于复用。
- 不使用 WorkBuddy、腾讯、CodeBuddy、OpenAI、Claude、GitHub 等官方 Logo 的仿制图。
- 不把生成式图片描述为官方素材。
- 记录生成 prompt、用途、限制和版权 / 商标注意事项。
