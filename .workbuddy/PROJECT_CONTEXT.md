# Project Context

## Project identity

- Repository: `workbuddy-best-practice`
- Subtitle: `From workplace prompting to agentic office automation`
- 中文定位：从办公提效到企业 Agent 工程化
- Nature: unofficial community best-practice repository

## Audience

- 企业职能人员：行政、人事、财务、市场、运营、销售支持、项目管理。
- AI Agent 实践者：希望把 prompt、workflow、Skill、Connector、Automation 变成可维护资产的人。
- 企业培训、知识库、IT、安全和数字化团队。

## Core methodology

```text
Idea → Task Brief → Context Package → Manual Run → Result Review → Workflow Template → Expert / Skill / Connector → Automation → Cloud Agent
```

核心迁移：把一次性办公 prompt 沉淀为可验收任务、可复用 workflow 和可治理的企业 Agent 资产。

## Source boundaries

- WorkBuddy 官方文档：可作为事实依据。
- 官方 Changelog：版本敏感内容的优先检查入口。
- Claude Code best practice：只能作为工程化组织方式参考，不是 WorkBuddy 官方事实来源。
- 第三方文章 / 社区帖子：只能标注为“社区观察”或“第三方评价”。

## Terminology

- WorkBuddy：全场景职场 AI 智能体桌面工作台。
- Task Brief：把模糊需求变成目标、输入、输出、边界、验收标准。
- Context Package：任务所需资料、规则、示例、术语、约束的组合。
- `.workbuddy/`：本仓库实践约定的项目级上下文目录，不是官方规范。
