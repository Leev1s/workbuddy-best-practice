# 会议记录整理为纪要和行动项

## 场景

会议结束后，团队手里有录音转写、聊天记录、零散笔记或腾讯会议相关材料，需要整理成正式纪要，并沉淀行动项。官方实践案例提到会议纪要整理，以及通过腾讯会议 Skill 进行会议预约、查询、录制、转写和 AI 纪要整理等操作。

## 任务目标

- 把原始记录整理为结构化会议纪要。
- 提炼关键结论、待办事项、负责人、截止时间和未决问题。
- 标记不确定信息，避免把模型推测写成会议事实。
- 生成可直接发给参会人的纪要版本和内部复盘版本。

## 输入材料

- 会议主题、日期、参会人和会议背景。
- 会议录音转写、聊天记录、手写笔记或会议文档。
- 项目背景资料、术语表、行动项格式要求。
- 发送对象：管理层、项目组、客户或内部留档。

## 推荐工作流

1. 准备一个会议工作空间，放入转写稿、聊天记录和参考资料。
2. 要求 WorkBuddy 先识别会议主题、议题结构和明显缺失信息。
3. 生成第一版纪要，明确区分“已确认结论”和“待确认事项”。
4. 人工补充负责人和截止时间，或要求 WorkBuddy 把缺失字段列为待确认。
5. 生成行动项表格，按负责人分组。
6. 如使用腾讯会议相关 Skill 或连接器，先确认授权范围和 Token 管理要求。

## 可复制 task brief

```markdown
请根据当前工作空间中的会议材料，整理一份正式会议纪要。

背景：
- 会议主题：{填写会议主题}
- 会议日期：{填写日期}
- 参会人：{填写参会人或角色}
- 纪要读者：{项目组/管理层/客户}

输入材料：
- 转写稿：{文件名}
- 聊天记录或补充笔记：{文件名}
- 相关背景资料：{文件名，可选}

输出：
1. `meeting-minutes.md`
   - 会议背景
   - 讨论议题
   - 关键结论
   - 行动项
   - 待确认问题
2. `action-items.csv`
   - 事项
   - 负责人
   - 截止时间
   - 依赖条件
   - 风险等级

规则：
- 不要编造未出现在材料中的结论。
- 无法确认负责人或截止时间时，写“待确认”。
- 涉及客户、财务、人事或合同内容时保留原始表述，不做夸张改写。
```

## 可能用到的 WorkBuddy 能力

- 上传文件、引用上下文和持续对话补充信息。
- 文档生成与二次修改。
- 腾讯会议 Skill 或相关连接能力，前提是当前环境已安装并完成授权。
- 任务列表中继续已有任务，保留会议上下文。
- 结果区预览纪要和导出文件。

## 验收标准

- 会议纪要包含背景、议题、结论、行动项、待确认问题。
- 行动项表格字段完整，负责人和截止时间缺失时明确标记。
- 没有把推测、建议或模型补全写成已发生事实。
- 读者版本语气合适，内部复盘版本保留风险和争议点。
- 所有引用的会议材料都来自本次工作空间或用户明确提供的上下文。

## 风险提醒

- 会议转写可能有识别错误，涉及决策、金额、责任人和截止时间的内容必须人工复核。
- 腾讯会议 Token 或个人凭证属于敏感信息，不应写入公开文档、项目记忆或模板。
- 如果纪要将发送给客户或管理层，先人工检查保密信息和未确认事项。
- 不要让自动化无人值守发送会议纪要，除非已经有明确审批流程和收件人规则。
