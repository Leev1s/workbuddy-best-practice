# 11-case-studies

本章收集可复制的 WorkBuddy 办公案例。案例来自 WorkBuddy 官方实践案例和官方文档列出的常见办公场景，再转写成更适合企业内部复用的 task brief、工作流和验收标准。

本章不是 WorkBuddy 官方案例镜像。所有实践建议都应以官方文档、当前产品界面和企业内部权限规则为准。

## 使用方式

- 先复制案例中的 task brief，再替换输入材料、输出格式、范围和验收标准。
- 第一次运行建议只处理小样本，确认命名规则、报告结构或推送内容正确后再扩大范围。
- 涉及文件写入、删除、脚本执行、连接器、自动化和第三方服务时，优先使用默认权限，并把输入材料放在独立工作空间。
- 每个案例都要保留人工验收环节，不把 WorkBuddy 的输出直接作为最终业务结论或自动发送内容。

## 案例选择原则

- 优先选择官方文档明确出现的实践类型：文件内容识别与处理、文档生成与编辑、数据分析并可视化、每日资讯简报、会议管理、腾讯文档管理。
- 对常见办公场景只做保守推导，不扩展成官方未说明的系统能力。
- 能力描述使用“可能用到”，避免暗示所有环境都已安装相同 Skill、Connector 或自动化配置。
