# 项目汇报文档与 PPT 生成

## 场景

项目负责人需要把项目总结材料、会议纪要、数据表和下一步计划整理成正式汇报文档或 PPT。官方实践案例提到 WorkBuddy 可生成 Word 文档、修改已有内容，也可根据素材和模板制作 PPT，并建议说明读者、页数、结构和风格偏好。

## 任务目标

- 把零散素材整理为一份结构完整的项目汇报。
- 根据受众生成管理层版、业务评审版或项目组版。
- 输出 Word / Markdown 文档和 PPT 大纲，必要时生成演示文稿。
- 保留数据来源和待确认信息，避免把不完整材料包装成确定结论。

## 输入材料

- 项目背景、目标、范围、周期和负责人。
- 里程碑、成果数据、风险清单、会议纪要和复盘材料。
- 参考模板、品牌规范或页数限制。
- 目标读者和使用场景：立项、周会、复盘、业务评审或管理层汇报。

## 推荐工作流

1. 先要求 WorkBuddy 读取素材并生成“材料清单 + 缺口清单”。
2. 确认汇报目标、读者、页数、风格和必须包含的章节。
3. 生成汇报文档初稿，人工检查事实、数据和措辞。
4. 基于确认后的文档生成 PPT 大纲，再生成 PPT。
5. 通过结果区检查产物、变更和预览，逐页确认标题、图表和结论。
6. 最后生成一页“讲稿提示”和“会前检查清单”。

## 可复制 task brief

```markdown
请根据当前工作空间中的项目材料，生成一份项目汇报文档和 PPT 大纲。

项目背景：
- 项目名称：{项目名称}
- 汇报对象：{管理层/业务评审/项目组/客户}
- 汇报目的：{立项/进展同步/复盘/争取资源}
- 时间范围：{起止时间}

输入材料：
- 项目总结：{文件名}
- 数据材料：{文件名}
- 会议纪要：{文件名，可选}
- 参考模板：{文件名，可选}

输出：
1. `project-report.md`：包含背景、目标、进展、成果数据、问题风险、下一步计划。
2. `ppt-outline.md`：10 页以内，每页包含标题、核心观点、建议图表或素材。
3. `speaker-notes.md`：每页 3-5 条讲稿提示。
4. `missing-info.md`：列出缺失数据和待确认表述。

写作要求：
- 面向汇报对象，语气专业克制。
- 所有数据结论必须能对应到输入材料。
- 不要虚构 ROI、客户反馈、收入影响或管理层评价。
- 对不确定信息标注“待确认”。
```

## 可能用到的 WorkBuddy 能力

- 文档生成、已有内容优化和多轮修改。
- PPT 生成或结合办公文档类 Skill 进行格式化输出。
- 上传文件、引用上下文和右侧结果区预览。
- 任务管理中的历史任务继续处理。

## 验收标准

- 文档结构完整，汇报目的和读者明确。
- 关键数据、项目状态和风险均能追溯到输入材料。
- PPT 大纲不超过约定页数，每页只有一个核心观点。
- `missing-info.md` 列出不能确认的数据、结论和素材缺口。
- 没有出现营销式夸大、未授权承诺或官方未提供的能力描述。

## 风险提醒

- 管理层汇报中的数据、预算、排期和风险等级必须由负责人复核。
- 如果使用模板或品牌素材，要确认版权和企业内部使用权限。
- 不要让 WorkBuddy 直接覆盖已有正式文档；先输出新版本或草稿。
- 涉及客户、合同、财务、人事内容时，避免把敏感信息放入公开模板。
