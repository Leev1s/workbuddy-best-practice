# 批量文件识别与归档

## 场景

团队把合同、票据、截图、会议材料、客户资料临时放进同一个文件夹，文件名混乱，后续检索困难。官方实践案例提到可让 WorkBuddy 根据文件内容或属性，按统一规则批量重命名，并建议先展示重命名预览结果。

## 企业价值与前置条件

- 企业价值：降低误命名、误归档和交接成本，让文件整理过程可审计。
- 适用：低风险文件副本、规则明确的批量命名、需要人工确认后再移动的归档。
- 不适用：合同原件、审计原件、客户隐私或缺少备份的唯一副本。
- 前置条件：独立工作空间、文件副本、命名规则、冲突命名处理规则、人工确认人。

## 任务目标

- 识别指定工作空间中的文件内容、类型或可用属性。
- 生成统一命名规则，例如 `日期-主题-类型-序号`。
- 先输出重命名预览表，再在人工确认后执行批量重命名或归档。
- 生成一份归档说明，记录规则、异常文件和未处理原因。

## 输入材料

- 一个独立工作空间，内含需要整理的文件副本。
- 命名规则：日期来源、主题提取方式、类型分类、序号格式。
- 不允许修改的文件清单，例如合同原件、审计材料、只读模板。
- 期望输出：预览表、重命名后的文件、归档说明。

## 推荐工作流

1. 新建独立文件夹，只放入待处理文件副本。
2. 在 WorkBuddy 创建任务时选择该文件夹作为工作目录。
3. 先要求 WorkBuddy 扫描文件并输出“原文件名、推测类型、建议新文件名、依据、风险备注”。
4. 人工检查命名规则和异常文件，明确是否执行重命名。
5. 执行前要求保留预览表；执行后检查结果区的产物、全部文件和变更。
6. 对无法识别或冲突的文件，单独建立 `needs-review` 清单，不强行改名。

## 可复制 task brief

```markdown
请整理当前工作空间中的文件副本，不要处理工作空间外的任何文件。

目标：
- 根据文件内容、文件类型和可用时间信息，生成统一文件名。
- 命名格式为：YYYYMMDD-主题-类型-序号。
- 先生成重命名预览表，不要直接执行重命名。

输入：
- 当前工作空间内的所有文件。
- 如无法判断日期，请标记为 unknown-date。
- 如无法判断主题，请标记为 needs-review。

输出：
1. `rename-preview.md`：包含原文件名、建议新文件名、判断依据、风险备注。
2. `needs-review.md`：列出不建议自动重命名的文件和原因。
3. `archive-plan.md`：建议归档目录、冲突处理和需要人工确认的移动操作。

执行限制：
- 不删除任何文件。
- 不覆盖同名文件。
- 不修改工作空间外文件。
- 等我确认预览表和归档方案后，再继续执行重命名或移动。
```

## 可能用到的 WorkBuddy 能力

- 自然语言任务创建。
- 工作目录读取和本地文件处理。
- 文件内容识别、批量处理和结果文件生成。
- 结果区中的产物、全部文件、变更查看。
- 默认权限下的高风险操作确认。

## 验收标准

- `rename-preview.md` 覆盖所有待处理文件。
- 每个建议文件名都能追溯判断依据。
- 未识别、冲突、敏感或不确定文件进入 `needs-review.md`。
- 未出现删除文件、覆盖原件或写入工作空间外路径。
- 人工确认后执行的文件名与预览表一致。
- 归档移动只发生在确认过的工作空间子目录内。

## 风险提醒

- 批量重命名会影响后续检索、合同归档和审计追踪，正式资料应使用副本试跑。
- 默认权限能在删除、工作空间外写入或脚本执行等高风险动作前请求确认，但不能替代备份。
- 如果 WorkBuddy 需要执行脚本或调用外部程序，应先要求解释脚本目的、影响范围和恢复方案。
- 不要把客户隐私、财务票据、合同原件直接放入未经授权的共享工作空间。
