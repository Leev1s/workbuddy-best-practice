# 业务数据分析与可视化报告

## 场景

运营、销售、财务或项目团队需要把 Excel / CSV 数据整理成图表和结论，用于周会、复盘或管理层汇报。官方实践案例提到可上传 Excel、CSV 等数据文件，说明指标、图表类型、统计维度和时间范围，生成图表和结论说明。

## 任务目标

- 读取指定数据文件并完成基础数据检查。
- 按业务问题生成统计结果、图表和解释。
- 输出可复核的数据处理说明，不隐藏缺失值、异常值和口径问题。
- 生成一份适合汇报的分析报告。

## 输入材料

- Excel / CSV 数据文件。
- 指标定义和统计口径，例如销售额、利润、转化率、留存率。
- 时间范围、分组维度、图表偏好和报告读者。
- 数据使用限制，例如脱敏要求、不可外发字段、禁止修改原始文件。

## 推荐工作流

1. 将原始数据副本放入独立工作空间，不直接处理生产系统导出的唯一文件。
2. 先要求 WorkBuddy 输出数据概览：字段、样本量、缺失值、异常值、可能的口径问题。
3. 人工确认分析口径，再执行统计和可视化。
4. 要求输出图表、指标表、结论摘要和数据处理说明。
5. 检查结果区的产物和预览，核对关键数字是否与原始数据一致。
6. 将“业务判断”和“数据事实”分开写，避免过度解释。

## 可复制 task brief

```markdown
请分析当前工作空间中的业务数据文件，并生成可视化报告。

业务问题：
- {例如：分析 2026 年 Q1 各产品线销售额、利润和增长趋势}

输入数据：
- 数据文件：{文件名}
- 时间范围：{时间范围}
- 关键指标：{指标列表}
- 分组维度：{月份/产品线/区域/渠道}

第一步：
- 先生成 `data-check.md`，说明字段、样本量、缺失值、异常值和口径疑问。
- 等我确认分析口径后，再继续生成图表和报告。

输出：
1. `analysis-report.md`：包含数据概览、主要发现、图表说明、业务建议、限制说明。
2. `metrics-table.csv`：关键指标汇总表。
3. 图表文件：按月份、产品线或指定维度输出。

约束：
- 不修改原始数据文件。
- 不夸大因果关系，只能基于数据描述趋势和相关观察。
- 对缺失值、异常值和样本量不足的问题必须单独说明。
```

## 可能用到的 WorkBuddy 能力

- 上传或引用 Excel / CSV 文件。
- 数据分析、统计汇总和图表生成。
- 文档生成与结果预览。
- 任务对话中的口径确认和二次修改。
- 默认权限下的工作空间文件读写。

## 验收标准

- `data-check.md` 明确列出字段、样本量、缺失值和异常值。
- 报告中的关键数字能在 `metrics-table.csv` 或原始数据中复核。
- 图表标题、坐标轴、单位和统计口径清晰。
- 结论区区分“数据事实”“可能解释”和“建议动作”。
- 原始数据未被覆盖或删除。

## 风险提醒

- WorkBuddy 可辅助分析和可视化，但业务口径、数据质量和最终决策仍需人工负责。
- 自动生成图表可能选择不合适的聚合方式，关键指标要人工复算。
- 涉及个人信息、客户数据或财务数据时，先脱敏并控制工作空间访问范围。
- 不要把相关性写成因果关系，不要根据样本不足的数据给出强结论。
