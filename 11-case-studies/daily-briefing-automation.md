# 每日资讯简报自动化

## 场景

团队希望每天固定时间收到天气、行业动态、重点新闻或任务摘要。官方实践案例提到可先连接 QQ 邮箱，手动触发一条简报确认内容和发送流程，再创建每日自动推送任务。官方自动化文档说明，自动化会按设定时间以当前登录身份发起 Agent 任务，并保存任务名称、prompt、调度规则、工作目录、执行状态等配置。

## 任务目标

- 设计一条可复用的每日简报 prompt。
- 先手动运行，确认信息来源、结构、收件人和发送效果。
- 再配置自动化任务，按固定时间生成并推送。
- 保留运行记录和人工抽检机制。

## 输入材料

- 简报主题：天气、AI 行业动态、公司新闻、项目任务摘要等。
- 信息来源约束：官方来源、指定网站、内部文档或用户提供材料。
- 收件人和推送渠道，例如 QQ 邮箱或 WorkBuddy 小程序。
- 调度规则：频率、生效日期、运行时段和失败处理方式。

## 推荐工作流

1. 先确认是否需要连接器，例如 QQ 邮箱；配置前阅读授权范围。
2. 手动发起一次简报任务，只生成草稿，不自动发送。
3. 人工检查来源、摘要准确性、敏感信息和收件人。
4. 再手动执行发送，确认收件和格式。
5. 在自动化页面创建任务，填写名称、工作空间、prompt、模型/技能、定时规则和推送设置。
6. 每周抽查自动化历史记录，检查失败、内容漂移和来源质量。

## 可复制 task brief

```markdown
请生成今日资讯简报草稿，先不要发送。

简报目标：
- 读者：{个人/团队/管理层}
- 主题：{例如：本地天气、AI 行业动态、重点新闻、我的任务摘要}
- 输出语言：简体中文
- 长度：800 字以内

信息来源规则：
- 优先使用官方或可信来源。
- 对无法确认的信息标注“待核实”。
- 不使用未经确认的小道消息。

输出结构：
1. 今日摘要：3-5 条。
2. 重点新闻：每条包含标题、来源、时间、摘要。
3. 行业动态：只保留与 {行业/团队} 相关的信息。
4. 待办提醒：如无可靠输入，写“未提供待办来源”。
5. 风险提示：列出可能需要人工核实的内容。

发送限制：
- 本次只生成 `daily-briefing-draft.md`。
- 等我确认后，再发送到 {邮箱/小程序/其他渠道}。
```

## 可能用到的 WorkBuddy 能力

- 自动化任务：按时间规则执行周期性任务。
- 连接器：例如 QQ 邮箱，用于发送、搜索或整理邮件，前提是完成授权。
- 信息检索、整理、文档生成和推送到小程序。
- 任务历史记录和执行状态查看。

## 验收标准

- 手动试跑至少一次，草稿内容和发送链路均通过人工确认。
- 简报结构稳定，来源、时间和待核实信息清楚。
- 自动化配置包含明确工作空间、prompt、调度规则和推送渠道。
- 收件人、授权账号和第三方凭证经过安全确认。
- 自动化运行后可在历史记录中追踪执行状态和结果。

## 风险提醒

- 自动化以当前登录身份触发任务，可能调用已授权的模型、工具、MCP 或连接器；启用前要确认权限范围。
- 邮箱连接器涉及读取、发送邮件等权限，授权时要检查具体权限，避免误发或访问不必要邮件。
- 新闻和行业动态可能过期或不准确，关键决策不能只依赖自动简报。
- 不要在自动化 prompt 中写入密码、Token、客户隐私或内部敏感数据。
- 建议先生成草稿再人工确认；无人值守发送只适合低风险、可撤回、收件范围清晰的场景。
