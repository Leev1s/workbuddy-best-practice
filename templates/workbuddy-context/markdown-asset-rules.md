# Markdown And Asset Rules

适用场景：统一公开文档、图片和资产说明的输出质量。  
风险等级：Low。

## Markdown

- 中文为主，必要英文术语保留。
- 每篇正文只保留一个 H1；后续按 H2 / H3 展开。
- 事实、实践解释、风险提醒尽量分开。
- 模板用 fenced code block 或清晰字段，方便复制。
- 内部链接使用相对路径。
- 不把 `.workbuddy/` 写成 WorkBuddy 官方目录规范。

## Assets

- 仓库级多媒体资产放在 `assets/`。
- README 首屏只保留必要视觉，不堆叠重复图片。
- 不仿制 WorkBuddy、腾讯、CodeBuddy、OpenAI、Claude、GitHub 等官方 Logo。
- 不把生成式图片描述为官方素材。
- 记录生成 prompt、用途、限制和版权 / 商标注意事项。
