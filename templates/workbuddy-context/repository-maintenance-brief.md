# Repository Maintenance Brief

适用场景：维护 `workbuddy-best-practice` 或类似公开知识库。  
风险等级：Medium。  
产出物：Markdown 文档、模板、导航、review report 或小范围资产更新。

```markdown
# Repository Maintenance Task Brief

## Goal

[写清本次维护目标。]

## Inputs

- Current repository files.
- User request:
- Official WorkBuddy docs:
- Benchmark or community references:

## Required Reading

1. `AGENTS.md`
2. `README.md`
3. `sources.md`
4. Related chapter README or article files
5. [optional context package files]

## Output

- Files to add or update:
- Navigation to update:
- Report or checklist to update:

## Boundaries

- Do not write official facts without official WorkBuddy source.
- Do not copy official docs in long passages.
- Do not treat `.workbuddy/` as an official WorkBuddy directory.
- Do not equate repository Markdown memory with WorkBuddy official Memory.
- Do not commit temporary logs, caches, private context, task artifacts, secrets, or customer data.

## Definition Of Done

- [ ] Markdown heading hierarchy is valid.
- [ ] Internal links resolve.
- [ ] New articles include Further Reading.
- [ ] Risk notes exist for permissions, files, Skill, Connector, Automation, Memory, model API, or Cloud Agent topics.
- [ ] Required checks have been run.
- [ ] Summary explains facts changed, risks handled, and remaining limits.
```
