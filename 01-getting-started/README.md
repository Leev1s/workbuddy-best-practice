# Getting Started

本章帮助用户把第一次 WorkBuddy 任务写清楚、跑完整、验收好。重点是工作目录、上下文引用、输出要求、权限边界和结果复核。

| Guide | Use it for |
|---|---|
| [task-prompt-template.md](task-prompt-template.md) | 把一句话需求改写成目标、输入、输出、限制和验收标准明确的 Task Brief。 |
| [result-review.md](result-review.md) | 在接受产物前检查结果区、全部文件、变更、预览、风险和分享范围。 |
| [WorkBuddy Context Convention](../docs/workbuddy-context-convention.md) | 理解本仓库为什么不默认创建 `.workbuddy/`，以及如何用普通文档组织 Context Package。 |

## Start Here

1. 为任务建立独立工作目录或工作空间，只放必要文件副本。
2. 用 [Task Brief Template](../templates/task-brief-template.md) 写清目标、输入、输出和禁止操作。
3. 在 prompt 中显式引用需要的文件、图片、链接或 Context Package。
4. 任务完成后用 [Result Review Checklist](../templates/review-checklist.md) 验收。
5. 涉及删除、覆盖、脚本、第三方服务、Full Access 或无人值守流程时，先做 [Security Checklist](../templates/security-checklist.md)。
