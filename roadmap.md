# Roadmap

## Purpose

`workbuddy-best-practice` 的路线图目标是把仓库从 MVP 骨架维护成可持续更新的办公 Agent 工程化知识库。路线图不承诺 WorkBuddy 官方能力，仅描述本社区仓库计划沉淀的内容。

## Phase 1: MVP foundation

- 建立 README、AGENTS、sources、changelog-watch。
- 完成 overview、任务 prompt、结果验收、权限模式和核心模板。
- 明确官方事实、实践解释、风险提醒的写作边界。

## Phase 2: Prompt patterns and workflows

- 补充 `02-prompt-patterns/`：办公任务、数据分析、PPT 报告、批量文件处理、深度研究、反模式。
- 补充 `03-workflows/`：Research → Plan → Execute → Review → Ship、文档生成、Excel 分析、会议纪要、周报、自检。
- 为每类 workflow 提供输入、输出、边界、验收标准和失败处理。

## Phase 3: Experts, Skills, Connectors

- 补充专家设计原则、专家团协作模式、Reviewer / Data Analyst / Office Automation 专家卡。
- 补充 Skill 概念、选择指南、官方优先原则、第三方 Skill 风险评估。
- 补充 Connector 概念、MCP + CLI 与 Skill + CLI 的方法论差异、腾讯文档 / 邮件 / 会议 / 企业系统集成流程。

## Phase 4: Automation and governance

- 补充 Automation 设计规范、每日资讯简报、周报自动化、定时文件处理、自动化风险 checklist。
- 补充文件删除、脚本执行、敏感数据和企业采用 checklist。
- 建立更新日志影响评估流程。

## Phase 5: Model, Memory, Cloud Agent and case studies

- 补充模型选择、BYOK、本地模型、Memory 最佳实践与风险。
- 补充 Cloud Agent 的概念、运行时、生命周期、checkpoint、发布部署。
- 补充 HR、财务、市场、研究助理、知识库问答等案例。

## Maintenance principles

- 优先更新高复用模板、权限安全、Skill / Connector 风险和企业落地内容。
- WorkBuddy 版本相关内容必须回看官方 Changelog。
- 新增案例要说明输入样例、操作边界、验收标准、风险处理，不写成泛泛故事。
